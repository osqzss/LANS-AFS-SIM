# Streaming I/Q Samples via a Pipe

AFS signals can be transmitted with LimeSDR without generating a large I/Q data file by passing the I/Q sample data generated by LANS-AFS-SIM directly from the `stdout` of `afs_sim` to the `stdin` of `limeplayer` through a pipe.

An example command is shown below:

```bash
afs_sim -t 90 - 2>nul | limeplayer -
```

Here, `-` is used as an option to select `stdout` or `stdin` instead of a file. The `2>nul` option redirects stderr to the null device to suppress console messages from `afs_sim` during streaming.
